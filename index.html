이상진

백엔드 · AI 개발자 · 국민대학교 소프트웨어학부

고등학교 때 웹을 파싱해 정보를 긁어오는 챗봇을 만들던 흥미가, 지금은 문서를 검색해 근거와 함께 답하는 RAG와 AWS 위의 AI 서비스로 이어졌습니다.

카카오톡 봇과 얼굴인식·머신러닝을 독학했고, 소프트웨어 특기자전형으로 국민대학교에 들어와 다섯 개의 팀 프로젝트에서 백엔드와 인프라를 맡았습니다. '검색해서 가져온다'는 한 가지 관심이 파싱에서 챗봇으로, 다시 RAG로 자연스럽게 자랐습니다.

기능을 완성하는 것만큼이나 "왜 이렇게 만들었는지"를 아키텍처·트러블슈팅 기록으로 남기는 것을 일의 일부로 여깁니다. 복잡한 클라우드·AI 기술을 혼자 아는 데서 멈추지 않고 남이 이해하고 쓸 수 있는 형태로 바꾸는 일에 흥미가 있습니다.

다음: career.log 에서 지나온 과정을 확인하세요.

## 대학에 오기 전 고등학교 · 독학

남이 시켜서가 아니라 궁금해서 직접 만들던 시기. 이 파싱·딥러닝 경험으로 소프트웨어 특기자전형에 합격했습니다.

  • 카카오톡 봇 JAVA · 웹 파싱 — 친구들과 즐길 미니게임(로또·성 쌓기)과, 웹을 파싱해 게임 캐릭터 정보·날씨·실시간 검색어를 알려주는 봇. split·replace·trim으로 원하는 정보만 추출하는 법을 처음 익혔습니다.
  • 얼굴인식 딥러닝 Python · OpenCV — 아나콘다·Haar Cascade로 얼굴 인식. Computer Vision의 Classification·Object Detection 개념을 직접 공부하며 딥러닝에 입문했습니다.
  • 학식 앱 Android · jsoup — jsoup로 학식표를 파싱해 스마트폰에서 바로 보는 앱. AsyncTask로 파싱 전후 동작을 제어하고 try-catch로 예외에 대비했습니다.
  • 머신러닝 강화학습 연구 Reinforcement Learning — 공룡게임 AI로 강화학습을 연구. 8세대 만에 1000점에 도달하며 잘못된 학습과 올바른 학습의 차이를 관찰했습니다.

## 트러블슈팅

70초 AI 생성 vs 60초 CloudFront 타임아웃
문제
gpt-image 생성이 약 70초로 CDN 오리진 타임아웃(60초)을 넘겨 매번 실패
해결
검증 직후 jobId를 202로 반환, 생성은 백그라운드, 본인 작업만 3초 간격 폴링으로 회수
결과
60초 벽을 넘겨 AI 생성이 타임아웃 없이 안정화
마감 스케줄러가 DB 페일오버 시 죽거나 중복 처리
해결
pg_try_advisory_lock 분산락으로 락 미획득 시 tick을 건너뛰고 connect 실패도 흡수
결과
DB 단절 시 크래시·마감 중복 처리 모두 방지
result
251 커밋(1위) / REST 79개 / 마이그레이션 45개 중 37개 작성 / Python 이미지 처리 툴 4종.

회고 — AI 기능의 난이도는 모델 호출이 아니라 타임아웃·비용·캐시·실패 같은 운영 제약을 구조화하는 데 있었습니다.
