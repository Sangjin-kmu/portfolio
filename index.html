문제의식

대학생 굿즈(과잠·키링·텀블러)는 수요가 분명한데도 디자인의 벽, 소량 제작 단가, 공동구매 운영의 번거로움에 막힙니다. 두띵은 이 셋을 하나의 흐름으로 잇습니다. 기술적으로 가장 까다로웠던 지점은 ① AI 이미지 생성이 한 번에 60초를 넘겨 CDN 타임아웃에 걸리는 것, ② 결제 마감 스케줄러가 상시 안전하게 돌아야 하는 것, ③ 흰 제품에 디자인을 입힐 실루엣 마스크가 부정확한 것이었습니다.

목표달성형 공동구매 펀딩

목표 금액이 모여야 제작·결제되는 리스크 제로 구조. 초기 자본 0원·재고 0으로 누구나 펀딩을 개설하고, 실시간 달성률·남은 시간·참여 인원을 투명하게 보여줍니다.

  • 60초 주기 결제 마감 스케줄러를 advisory-lock으로 보호
  • 서버측 가격 재계산으로 위·변조 방어

트러블슈팅
외부 경로 API 실패 시 경로가 잠긴 채 회복 수단이 없어 알림 누락
문제
ODsay가 일시 장애(5xx)나 키 만료(401/403)로 실패하면 경로가 PENDING_RETRY로 잠겨, 사용자가 직접 수정하지 않으면 영구 정체
해결
실패를 transient/auth/mapping으로 분류하는 RouteRefreshOutcome enum + 스케줄러가 5분 쿨다운·최대 5회 배치 재시도, 401/403은 운영자에 노출
결과
사용자 개입 없이 일시 장애가 약 25분 내 자동 복구. 단위 12건 + 통합 회귀 가드
데모 직전 500 — 요일 6개 이상이면 일정 등록 실패
원인
요일 CSV 컬럼이 VARCHAR(20). 평일 5요일(19자)은 통과하지만 6요일 이상은 길이를 넘겨 'Data too long' — '요일 개수'가 진짜 트리거
해결
마이그레이션으로 VARCHAR(31)(MySQL INSTANT 무중단) + DTO @Size(max=7) 검증을 함께 적용 (defense in depth)
결과
6요일 이상 정상 등록. 통합 테스트 3건으로 재발 차단
추가로 만든 것 · 개발자 유형 테스트
