KDD — 학사 규정 RAG 챗봇 백엔드
2026.03~06 · 백엔드 2인 중 1인 · Java, Spring Boot 3.4, PostgreSQL, PDFBox, WebFlux · GitHub

학칙·학사 규정 PDF를 근거로 출처와 함께 답하는 RAG 챗봇입니다. AI가 지어내지 않고 실제 문서를 인용하게 만드는 것이 목표였고, 저는 백엔드를 맡았습니다.

Next.js 웹 → Spring 백엔드(담당) → FastAPI AI 서버 문서 업로드·파싱·청킹과 FAQ·통계, AI 서버 오케스트레이션을 담당. 임베딩·LLM 자체(AI 서버)는 팀원이 맡았습니다.

주요 역할

  • 문서 파이프라인 전체 구현 — PDF 업로드 → PDFBox 페이지별 파싱 → 페이지 경계를 넘지 않는 청킹(550자·100자 오버랩) → AI 서버 임베딩 요청
  • AI 서버 연동 3종(임베딩·삭제·FAQ 분석)을 클라이언트로 캡슐화하고 트랜잭션 경계를 재설계
  • FAQ 자동 인입 파이프라인 — 질문 수집 → AI 분석 → PII(학번·이메일·전화) 마스킹 → 관리자 검수 → 등록
트러블슈팅 · AI 호출이 DB 트랜잭션을 붙잡는 문제
문제 수십 초짜리 임베딩 호출이 트랜잭션 안에서 커넥션·락을 점유해 동시 업로드 시 커넥션 풀이 마를 위험
해결 영속성 로직을 별도 트랜잭션으로 분리, 트랜잭션이 닫힌 뒤 AI 호출. saveAndFlush로 PK 선확보
